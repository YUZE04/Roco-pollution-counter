# 🎬 污染计数器 v1.2.3 · 使用教程录屏脚本

> 目标时长：**4 ~ 5 分钟** · 快节奏 · 每段 8~20 秒
> 建议分辨率：1920×1080 · 录屏帧率：30 / 60
> 作者：小丑鱼 · 抖音号 `conflicto834`

## 🎙️ 第 1 幕｜开场（0:00 – 0:15）

**画面**：桌面特写，图标居中 + 标题卡  
**操作**：显示项目图标 + "污染计数器 v1.2.3"  
**旁白**：

## 📦 第 2 幕｜下载与解压（0:15 – 0:40）

**画面**：浏览器打开 GitHub Release 页 → 下载 zip → 右键解压  
**操作**：

1. 点进 GitHub Release 页
2. 下载 `污染计数器v1.2.3.zip`（约 300 MB）
3. 解压到 **非中文、非 Program Files** 的目录，例如 `D:\Tools\污染计数器`

**旁白**：

> 去我 GitHub 主页下最新版，解压别放 C 盘 Program Files，也尽量避开中文路径——这样能少踩坑。

💡 **强调**：把"**以管理员身份运行**"的右键菜单特写一下。

---

## 🚀 第 3 幕｜首次启动（0:40 – 1:05）

**画面**：双击 exe → UAC 弹窗点 **是** → 程序启动 → 展示主窗口 + 悬浮窗同时出现  
**操作**：

1. 右键 exe → **以管理员身份运行**
2. UAC 确认
3. 等 OCR 初始化（状态栏会显示 **OCR 就绪**）

**旁白**：

> 一定要管理员身份启动，不然热键抓不到。首次打开会加载 OCR 模型，大概等 3 秒。右边那个半透明的小框，就是悬浮窗，一会儿我们让它飞到游戏画面上。

---

## ⚙️ 第 4 幕｜分辨率 + 区域校对（1:05 – 1:35）

**画面**：主窗口 → **设置** 标签页 → 分辨率下拉菜单  
**操作**：

1. 点开 **设置** 标签
2. 下拉选你屏幕对应的预设（常用 `2560x1600_150缩放` / `1920x1080`）
3. 点 **预览识别区域**，确认红框刚好框住游戏中的"污染提示"文字

**旁白**：

> 这一步最关键——**分辨率选错 OCR 就瞎了**。如果你的屏幕和预设都不一样，可以点下面的"预览识别区域"，看红框有没有对准游戏里那行关键字，有偏差就手动微调坐标。

---

## ▶️ 第 5 幕｜开始监测（1:35 – 2:15）

**画面**：回到 **状态** 标签 → 点 **开始监测** → 切到游戏窗口  
**操作**：

1. 点 **开始监测**（按钮变红，悬浮窗左上角绿灯亮）
2. Alt+Tab 切到游戏
3. **播放提前录好的 30 秒游戏片段**
4. 镜头对着悬浮窗，展示计数自动 +1

**旁白**：

> 按下开始监测，悬浮窗的灯变绿，这时候只要游戏里触发污染事件——我这条示例里大概每 15 秒来一次——悬浮窗的数字就会自动加 1。它识别完有 12 秒冷却，**不会因为文字停留而重复计数**。

---

## 🪟 第 6 幕｜悬浮窗 + 锁定（2:15 – 2:40）

**画面**：拖动悬浮窗 → 右键菜单 → 锁定 → 鼠标穿透演示  
**操作**：

1. 按住悬浮窗拖到游戏右上角
2. 右键 → **锁定悬浮窗**（灯变成 🔒）
3. 把鼠标移到悬浮窗上 **点击游戏**，演示点击能穿过去

**旁白**：

> 悬浮窗可以拖到你看着舒服的位置。右键锁定之后，**整个小窗变成鼠标穿透的**，你在游戏里点技能、拖宠物都不会戳到它。想解锁就按 `-` 键。

---

## ⌨️ 第 7 幕｜热键（2:40 – 3:05）

**画面**：主窗口 **设置** → 热键表格 → 高亮四行  
**操作**：简单在屏幕上显示下面四行

| 热键 | 作用 |
|:-:|:--|
| `7` | ⏯️ 暂停 / 继续 |
| `8` | ➕ 手动 +1 |
| `9` | ➖ 手动 -1 |
| `-` | 🔒 锁定 / 解锁 |

**旁白**：

> v1.2.3 起热键简化了——一个键搞定暂停继续，启动还是走主窗口按钮。`8` `9` 是手动加减，OCR 漏识别了自己补；锁定用 `-`。全部能自定义。

💡 演示一次 `8` 按下 → 悬浮窗数字 +1；再按 `9` → -1。

---

## 📊 第 8 幕｜统计 + 历史数据（3:05 – 3:30）

**画面**：主窗口 **统计** 标签 → 切换日期 → 按精灵统计表  
**操作**：

1. 切到 **统计** 标签
2. 滚动按日期列表
3. 点按精灵列表，演示条目可排序、可右键编辑

**旁白**：

> 所有数据都自动按天、按精灵双维度存下来。哪天刷了多少、刷了哪些精灵一目了然。错了也能右键直接改数字，不用去翻 JSON。

💡 **加分演示**：展示一次"导入旧版 `pollution_count.json`"——点设置里的导入按钮，选一个旧文件。

---

## 🧩 第 9 幕｜OCR 纠错（3:30 – 3:50）

**画面**：打开 `pollution_config.json` 文本  
**操作**：高亮 `ocr_name_aliases` 字段

```json
"ocr_name_aliases": {
  "曙光瑜瑜": "噬光嗡嗡"
}
```

**旁白**：

> 有些精灵名字 OCR 会认错，比如"噬光嗡嗡"老被读成"曙光瑜瑜"。v1.2.3 默认带了一条修正，你也可以自己加——配置文件里的 `ocr_name_aliases`，左边写错的，右边写对的，重启生效。
